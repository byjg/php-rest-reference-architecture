Quick Start Guide

Six common tasks, collapsed by default — open just the one you need.

Step 1: Create an "up" migration file in db/migrations/up/ using format 0000X-description.sql
Step 2: Create a "down" migration file in db/migrations/down/ for rollbacks (number should be X-1)
Step 3: Apply the migration
APP_ENV=dev composer run migrate -- update
Reset database: Destroys all data and reapplies migrations
APP_ENV=dev composer run migrate -- reset --yes
Step 1: Ensure your table exists (via migrations)
Step 2: Run codegen for Repository Pattern (clean separation of concerns)
APP_ENV=dev composer run codegen -- --table=products all --save
Alternative: Use ActiveRecord Pattern for rapid development
APP_ENV=dev composer run codegen -- --table=products all --activerecord --save
Step 3: Update OpenAPI documentation
APP_ENV=dev composer run openapi
Generated: Model, Repository/ActiveRecord, Service (repository only), REST Controller, Tests, and DI bindings
Access Swagger UI: View interactive API documentation
http://localhost:8080/docs
Update after code changes: Regenerate OpenAPI spec
APP_ENV=dev composer run openapi
OpenAPI file: public/docs/openapi.json
Step 1: Register in DI container (e.g., config/dev/05-services.php)
use ByJG\Config\DependencyInjection as DI; return [ ProductsService::class => DI::bind(ProductsService::class) ->withInjectedConstructor() ->toSingleton(), ];
Step 2: Get service from container in your code
use ByJG\Config\Config; $service = Config::get(ProductsService::class); $result = $service->getOrFail($id);
Note: Codegen automatically creates DI bindings. Uses PSR-11 compatible container.
Step 1: Create REST controller with OpenAPI attributes
use OpenApi\Attributes as OA; use ByJG\Config\Config; #[OA\Get( path: "/products/{id}", security: [["jwt-token" => []]], tags: ["Products"] )] #[RequireAuthenticated] public function getProduct(HttpResponse $response, HttpRequest $request): void { $service = Config::get(ProductsService::class); $result = $service->getOrFail($request->param('id')); $response->write($result); }
Step 2: Regenerate OpenAPI documentation
APP_ENV=dev composer run openapi
Tip: Use codegen to auto-create full CRUD endpoints
Login to get JWT token:
curl -X POST http://localhost:8080/login \ -H "Content-Type: application/json" \ -d '{"username":"admin@example.com","password":"!P4ssw0rdstr!"}'
Use token in requests:
curl http://localhost:8080/products/1 \ -H "Authorization: Bearer YOUR_TOKEN"
Protect endpoints: Use #[RequireAuthenticated] or #[RequireRole(User::ROLE_ADMIN)]
Access user data: JwtContext::getUserId(), JwtContext::getRole()
Default users: admin@example.com / user@example.com (password: !P4ssw0rdstr!)
